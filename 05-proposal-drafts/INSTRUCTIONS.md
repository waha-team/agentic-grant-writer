# Stage 05: Proposal Drafts

> **Your job:** Write complete draft proposals for each foundation on the shortlist, tailored to their specific requirements and interests.

---

## Before You Start

1. Read `CONTEXT/OUR_ORGANIZATION.md` — your source for organizational narrative
2. Read `CONTEXT/IMPACT_DATA.md` — your source for evidence and stories
3. Read `CONTEXT/STYLE_GUIDE.md` — your writing guide (READ THIS CAREFULLY)
4. Read the project subfolder's `PROJECT_BRIEF.md`, `ALIGNMENT_ANALYSIS.md`, and `APPLICATION_PREP.md`

## What to Do

For each project subfolder found in this directory:

### Step 1: Prioritize
Look at APPLICATION_PREP.md. Start with:
1. Foundations with the nearest deadlines
2. Foundations with the highest alignment scores
3. Foundations with the simplest applications (quick wins build momentum)

### Step 2: Write Each Proposal
For each foundation in APPLICATION_PREP.md, create a subfolder inside the project folder named after the foundation (e.g., `kern-family-foundation/`). Inside that subfolder, create:

#### `PROPOSAL_DRAFT.md`
The full narrative proposal, structured according to the foundation's requirements (from APPLICATION_PREP.md). If the foundation doesn't specify a structure, use:

1. **Executive Summary** (1 paragraph)
   - Who we are, what we're asking for, why it matters, how much
   
2. **Statement of Need** (1-2 paragraphs)
   - The problem this project addresses
   - Use data: unreached people groups, language access gaps, specific population needs
   - Make it vivid with a story from IMPACT_DATA.md if relevant
   
3. **Project Description** (2-4 paragraphs)
   - What Waha will do with this funding
   - Specific activities, timeline, deliverables
   - How this connects to Waha's broader mission
   
4. **Evidence of Effectiveness** (1-2 paragraphs)
   - Results from similar work Waha has done
   - Specific numbers: users, groups started, multiplication data
   - Testimonial or case study
   
5. **Budget Narrative** (1 paragraph + table)
   - What the money will be spent on and why each line item matters
   - Budget table from PROJECT_BRIEF.md, adjusted for this specific ask amount
   
6. **Organizational Capacity** (1-2 paragraphs)
   - Why Waha is the right team
   - Track record: 41+ languages, 110+ countries, 100K+ users
   - Team expertise
   
7. **Evaluation & Reporting** (1 paragraph)
   - How we'll measure success
   - What we'll report back to the funder
   
8. **Sustainability** (1 paragraph)
   - How the work continues after the grant period

**CRITICAL WRITING GUIDELINES (from STYLE_GUIDE.md):**
- Lead with story, back with data
- Use "facilitate" not "lead/teach"
- Use "restricted-access nations" not "dangerous places"
- Never name specific believers or locations in security-sensitive regions
- Tailor framing to the funder type (see OUR_ORGANIZATION.md "How to Frame Waha" section)
- Keep it as short as the funder allows — don't pad
- Be specific: "30 groups multiplied to 90 in 50 days" not "significant growth"

#### `BUDGET_DRAFT.md`
A formatted budget for this specific grant ask. Derived from PROJECT_BRIEF.md but:
- Adjusted to match the grant ask amount (which may be a portion of total project cost)
- Formatted to match the foundation's requirements if specified
- Includes brief justification for each line item

#### `COVER_LETTER_DRAFT.md` (if applicable)
A brief cover letter for foundations that accept mailed or emailed applications:
- Addressed to the specific contact from CONTACTS.md
- 3-4 paragraphs max
- References any prior connection or shared network
- Expresses enthusiasm for the foundation's specific mission
- Clear ask: what we're requesting and for what purpose

#### `LOI_DRAFT.md` (if the foundation requires a Letter of Inquiry first)
A 1-2 page letter of inquiry:
- Brief organizational overview
- Project summary
- Budget range
- Request for invitation to submit full proposal

### Step 3: Foundation-Specific Questions
If APPLICATION_PREP.md lists specific questions from the foundation's application, answer each one directly in a separate file called `APPLICATION_ANSWERS.md`.

### Step 4: Quality Check Your Own Work
Before moving on, verify each draft against:
- [ ] Does it address the foundation's stated priorities?
- [ ] Is the framing appropriate for this funder type?
- [ ] Are impact numbers accurate (per IMPACT_DATA.md)?
- [ ] Is the budget consistent with PROJECT_BRIEF.md?
- [ ] Are there any security-sensitive details that shouldn't be included?
- [ ] Is it within any stated word/page limits?
- [ ] Does it follow the STYLE_GUIDE.md?

### Step 5: Move Forward
After drafting proposals for all recommended foundations, move the project subfolder to `06-review/`.

## Decision Points

**Default to momentum.** Write the best draft you can and keep moving. The human review in Stage 06 is the safety net. But flag these:

- If the foundation asks a question you genuinely can't answer from available context → Put "[JOSH: PLEASE ANSWER]" inline in the draft and add to HUMAN_TODO.md: "[Foundation] application asks [question]. I don't have this information. Please answer directly in the draft."
- If a foundation requires a very different framing than others (e.g., secular development angle) → Note this prominently at the top of PROPOSAL_DRAFT.md so reviewers pay extra attention.
- If the budget doesn't fit the foundation's typical range → Note the mismatch and suggest adjusting the ask amount.

## Quality Standards

- Every proposal must be specifically tailored to the foundation. No generic copy-paste across proposals.
- Every proposal must include at least one specific data point and one human story.
- Every proposal must clearly state the ask amount.
- Budget math must add up.
- Tone must match the funder (see STYLE_GUIDE.md and OUR_ORGANIZATION.md framing guide).
