# Stage 00: Projects

> This folder holds project briefs — the initiatives that need grant funding. Each project gets its own subfolder with a PROJECT_BRIEF.md file.

---

## How This Stage Works

This is the **entry point** for the pipeline. Humans create project briefs here. When a project is ready to move forward, its entire folder is moved into `01-prospect-research/`.

## For Humans (Josh / Alycia / Valeria)

1. Copy the `_TEMPLATE/` folder and rename it to your project (e.g., `waha-farsi-localization/`)
2. Fill in the `PROJECT_BRIEF.md` template with as much detail as you can
3. When ready, move the folder to `01-prospect-research/`

## For Claude Code

If asked to help create a project brief:
1. Read `CONTEXT/OUR_ORGANIZATION.md` for overall context
2. Ask the human what initiative they want to fund
3. Generate a draft `PROJECT_BRIEF.md` using the template structure
4. Save it in a new subfolder here in `00-projects/`

## What a Good Project Brief Contains

At minimum:
- A clear project name and one-sentence description
- Who it serves (specific population, geography, language)
- What will happen (activities and deliverables)
- How much it costs (budget range)
- What success looks like (measurable outcomes)
- Timeline

The more specific and quantified the brief is, the better the prospect research and proposals will be.
